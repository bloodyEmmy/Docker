#flashcards
***
Интерфейс MACVLAN создается путем присоединения к физическому интерфейсу и имеет собственный MAC- и IP-адрес.
***
***Важные замечания***:
1. Интерфейс MACVLAN необходимо сопоставить с родительским интерфейсом. Для доступа к внешним сетям будет использоваться шлюз хоста.
2. Драйвер ([[Собственные (встроенные) сетевые драйверы Docker]]) работает только на хосте Linux и не поддерживается в [[Docker]] Desktop для Mac, Windows.
3. Ядро 4.0+
4. Назначен той же подсети, что и родительский интерфейс.
5. Для каждого родительского интерфейса в одной подсети можно определить только одну сеть типа MACVLAN.
6. Можно указать диапазон IP-адресов для [[Контейнер]], а также вспомогательный адрес, который [[IP Adress Management (IPAM)]] ***не должна*** назначать контейнерам.
